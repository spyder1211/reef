# MySQL クライアント（TablePlus風 Mac アプリ）設計書

- 作成日: 2026-06-06
- ステータス: ドラフト（v1 設計合意済み）

## 1. 背景（Context）

TablePlus の無料版は**同時に開けるタブが2つまで**に制限されており、複数テーブル／クエリを並行して扱う日常業務では不便。これを解消し、**タブを無制限に開ける** MySQL クライアントを内製する。あわせて、自分だけでなく**社内のメンバーにも配布**して使ってもらえる、使いやすいデスクトップアプリを目指す。

弊社で利用する RDB はほぼ MySQL のみのため、**MySQL 専用**として割り切ることで設計をシンプルに保つ。

## 2. 目的とゴール

- **主目的**: TablePlus 風のデータ閲覧 / SQL 編集 / カラム単位編集を、タブ無制限で快適に行える Mac アプリを作る。
- **差別化ポイント**: **タブ数無制限**。
- **配布**: macOS 向け `.app` / `.dmg` として社内配布できる。
- **重視点**: 実用性・安定性・誰でも使いやすい UI（個人＋社内ツール）。

### 非ゴール（Non-goals）

- MySQL 以外の DB 対応（PostgreSQL 等）は対象外。
- v1 ではスキーマ／テーブル構造の GUI 編集（テーブル/カラム作成・変更）は作らない（v2 以降）。
- 一般公開・App Store 配布は当面想定しない（Apple 署名/公証は任意・後付け）。

## 3. 対象ユーザー

- 自分（開発者）および社内メンバー。MySQL を日常的に閲覧・編集する人。
- 全員 macOS 利用。各自が自分の端末にアプリをインストールして使う。

## 4. スコープ（v1）

### コア機能

1. **接続管理**: MySQL 接続プロファイルの登録・保存。パスワードは macOS Keychain に保存。接続ごとの**色ラベル**（例: 本番=赤）。
2. **データ閲覧**: テーブル一覧（スキーマツリー）、スプレッドシート風グリッド、カラムソート。
3. **カラム別 複数フィルタ**: グリッド上部のフィルタバーで `[カラム][演算子][値]` の条件を複数追加し、AND/OR で結合。生成 WHERE 句をプレビュー。
4. **無制限タブ**: TableTab / SqlTab を制限なく開ける。再起動時にタブを復元。
5. **SQL エディタ**: シンタックスハイライト、スキーマ連動オートコンプリート、「カーソル位置の文／選択範囲／すべて」実行、複数結果タブ、クエリ履歴。
6. **カラム/セル単位のインライン編集**: セルを直接編集 → ステージング → 変更 SQL プレビュー → トランザクションでまとめてコミット。Undo（破棄）可能。

### 追加機能

7. **SSH トンネル接続**（踏み台サーバー経由）＋ SSL 接続。
8. **エクスポート/インポート**: CSV / JSON / SQL。**現在のフィルタ＆ソートに連動した CSV エクスポート**（絞り込み結果をそのまま出力。全件出力も選択可）。
9. **安全機能**: 読み取り専用モード（書き込み系をブロック）、本番（赤ラベル）接続はコミット前に確認、接続色ラベル。
10. **接続設定の社内共有**: 接続プロファイルをファイルにエクスポート／インポート（**パスワードは含めない**）。

### v2 以降（先送り）

- スキーマ／テーブル構造の GUI 編集（テーブル・カラム・インデックス・制約の作成/変更）。
- Apple コード署名／公証。
- プラグイン機構、内部ダッシュボード等。

## 5. アーキテクチャ

### プロセスモデル（Electron セキュリティ構成）

DB 認証情報を安全に扱うため、Electron 推奨の分離構成を採用する。

- **メインプロセス（Node.js）**: DB 接続・SQL 実行・SSH トンネル・Keychain・ファイル I/O など**特権操作を集約**。
- **レンダラ（UI）**: `contextIsolation: true` / `nodeIntegration: false`。DB や Node に直接触れない。
- **preload ブリッジ**: `contextBridge` で型付きの狭い API（例: `window.api.query()`）のみ公開し、`ipcRenderer.invoke` 経由でメインに依頼する。

### メイン側モジュール

| モジュール | 責務 |
|---|---|
| `ConnectionManager` | 接続プール（`mysql2`）管理、SSH トンネル（`ssh2`）／SSL 確立 |
| `ProfileStore` | 接続プロファイルの JSON 永続化、パスワードは Keychain（`keytar`）。エクスポート/インポート時はパスワード除外 |
| `QueryService` | SQL 実行（複数文・カーソル位置実行対応）、読み取り専用モードの強制 |
| `EditService` | ステージング編集 → 主キーを用いたパラメータ化 UPDATE/INSERT/DELETE 生成 → プレビュー → トランザクションでコミット |
| `FilterBuilder` | フィルタ条件 → パラメータ化 WHERE 句（プレビュー表示と安全な実行） |
| `ExportService` | CSV/JSON/SQL 出力。フィルタ＆ソート連動の CSV ストリーム出力 |
| `SchemaService` | DB/テーブル/カラム一覧の取得（ツリー表示・補完候補に利用） |

### レンダラ（UI）構成

- **アプリシェル**: 左サイドバー（スキーマツリー）＋ タブ作業領域 ＋ 下部ステータスバー ＋ 上部ツールバー。
- **タブシステム**: `TableTab` / `SqlTab` を無制限に保持。状態はストア（Zustand）で管理。再起動時にタブを復元。
- **データグリッド**: 大量データ向けに**仮想スクロール**（TanStack Table + TanStack Virtual）。型に応じたセルエディタ（テキスト/日付/JSON/NULL）。
- **フィルタバー**: 複数条件 UI。WHERE 句プレビュー。
- **SQL エディタ**: CodeMirror 6（SQL ハイライト＋スキーマ連動補完）。実行バー（カーソル位置/選択/すべて）、結果は複数タブ保持、クエリ履歴。

### 画面レイアウト

- 上: ツールバー（接続スイッチャ／色ドット、新規タブ、実行、SQL エディタ、エクスポート、読み取り専用トグル）。
- 上段: タブバー（⚡=SQL タブ、その他=テーブルタブ。無制限）。
- 左: スキーマツリー＋テーブル検索。
- 中央: テーブルタブ=フィルタバー＋編集可能グリッド／SQL タブ=エディタ（上）＋結果グリッド（下、比率ドラッグ可）。
- 下: ステータスバー（行数・実行時間・未保存件数・[コミット]/[破棄]）。

## 6. 主要データフロー

### セル編集 → コミット

1. ユーザーがグリッドのセルを編集 → レンダラが変更（行の主キー・カラム・旧値・新値）をタブ状態にステージング。
2. ステータスバーに未保存件数を表示。
3. **コミット**時、レンダラが全ステージング変更をメインへ送信 → メインが主キーを用いた**パラメータ化 UPDATE/INSERT/DELETE** を生成し、変更 SQL をプレビュー表示。
4. ユーザー確認後、**1 トランザクション**で実行 → 成功でステージングを確定・該当行を再取得。失敗時はロールバックしステージングを保持、失敗した文を提示。
- **エッジケース**: 主キーが無いテーブルは全カラム一致で更新を試み、一意に特定できない場合は警告して編集を拒否。

### フィルタ → 表示 → CSV エクスポート

1. フィルタバーの条件 → `FilterBuilder` がパラメータ化 WHERE 句を生成（プレビュー表示）。
2. メインが `SELECT ... WHERE ... ORDER BY ...` を実行し、ページング取得。
3. CSV エクスポート時は**同じフィルタ＆ソート条件**でメインが結果をストリーム取得し CSV へ書き出し（全件選択も可）。

### SQL 実行（カーソル位置）

- レンダラが対象の SQL 文（カーソル位置／選択範囲／全文）を切り出してメインへ送信 → 実行 → 結果を該当結果タブへ。

## 7. セキュリティ・安全性

- すべて**パラメータ化クエリ**（フィルタ・編集の SQL インジェクション防止）。
- `contextIsolation` 有効・`nodeIntegration` 無効・preload で最小 API のみ公開。
- パスワードは Keychain 保存。プロファイル JSON や共有エクスポートには**含めない**。
- **読み取り専用モード**: INSERT/UPDATE/DELETE/DDL をメインの `QueryService` でブロック（UI でも抑止）。
- 本番（赤ラベル）接続では破壊的コミット前に確認ダイアログ。

## 8. エラー処理

- 接続エラー（タイムアウト／認証失敗／SSH 失敗）: トースト＋タブのエラー状態、再試行導線。
- クエリエラー（構文・制約違反）: 結果ペインに MySQL のエラーコード／メッセージを表示。
- コミット失敗: トランザクションをロールバックし、ステージングを保持。どの文で失敗したか提示。
- エクスポート失敗: ファイル書き込みエラーを通知。

## 9. 技術スタック

- 実行基盤: **Electron**（`.app`/`.dmg`、`electron-builder` でパッケージング）。
- 言語: **TypeScript**。
- フロント: **React + Vite**、状態管理 **Zustand**。
- グリッド: **TanStack Table + TanStack Virtual**。
- SQL エディタ: **CodeMirror 6**。
- DB/接続: **mysql2**、SSH: **ssh2**、Keychain: **keytar**。
- テスト: **Vitest**（単体）、Docker MySQL（結合）、**Playwright for Electron**（E2E）。

## 10. テスト戦略（TDD）

- **単体（Vitest）**: `FilterBuilder`（条件→WHERE）、`EditService` の編集 SQL 生成、CSV シリアライズ、`ProfileStore` のエクスポート（パスワード除外を検証）。
- **結合**: Docker 上の MySQL に対し `QueryService` / `EditService`（接続、クエリ、編集コミットのトランザクション、読み取り専用強制）。
- **E2E（Playwright for Electron）**: 接続 → テーブル表示 → 複数フィルタ適用 → セル編集 → コミット → CSV エクスポート。SQL タブのカーソル位置実行。

## 11. パッケージング・配布

- `electron-builder` で macOS 向け `.dmg` を生成。社内配布は未署名でも可（初回起動時の Gatekeeper 警告は右クリック→開くで回避）。
- 将来的に Apple Developer 署名／公証を追加すれば警告なしで配布可能（v1 必須ではない）。
